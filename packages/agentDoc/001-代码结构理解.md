## 二、数据流与核心概念

### 2.1 数据结构（MusicScore）

- **MusicScore**：顶层，包含 `grandStaffs`、`affiliatedSymbols`（跨音符/小节符号如连音线、volta）、`width/height`、`bpm` 等
- **GrandStaff**：复谱表，包含多个 `SingleStaff`
- **SingleStaff**：单谱表，包含 `measures`
- **Measure**：小节，包含 `notes`、`barline`、`clef_f/b`、`keySignature_f/b`、`timeSignature_f/b`、`affiliatedSymbols`
- **NoteSymbol**（五线谱）：`voicePart1` / `voicePart2`（双声部），内含 `notesInfo`（region、accidental）、`chronaxie`、`beamType`、`augmentationDot` 等
- **NoteNumber**（简谱）：`voicePart`，`notesInfo` 为 `NotesNumberInfo`（syllable 0–7/'X'）

所有可选中元素均有 `id`，用于 VDom 的 `targetId` 映射和编辑时定位。

### 2.2 渲染流程

1. **props.data**（`MusicScore`）传入 `music-score` 组件
2. **musicScoreToVDom**：根据 `data.type`（StandardStaff / NumberNotation）选择五线谱或简谱实现，将 `MusicScore` 转为 `VDom[]`
3. **VDom**：虚拟节点，含 `tag`（grandStaff/singleStaff/measure/noteHead/rest/clef/barline/slot 等）、`targetId`、`skinKey`、`skinName`、`x/y/w/h`、`special`（slur/volta/beam 等）
4. **Group 组件**：根据 `skinKey` 从皮肤包取 SVG 内容，按 `node.w/h` 替换尺寸，注入 `data-target-id`
5. **slot**：插槽（g/g-l/g-r/s/m 等），通过 `slotConfig` 控制宽高，扩展可在插槽内渲染歌词等

### 2.3 选中与高亮

- 元素通过 `data-target-id`、`data-tag` 标记，便于点击识别
- `renderTest.vue` 已有 `handlePointerDown`：点击 slot 或符号时设置 `highlightTargetId`，并通过 `msRef.updateVDom` 将对应节点 `skinName` 改为 `red`，实现高亮
- **updateVDom**：`defineExpose` 的 `(updater) => updater(vDom.value)`，直接修改 vDom 引用，触发响应式更新

### 2.4 事件

- `pointerdown` / `pointermove` / `pointerup`：由父组件（如 renderTest）监听，目前仅 down 有逻辑
- `reRender`：vDom 重新计算后 emit，可用于扩展同步

## 三、编辑扩展规划（功能.md 摘要）

五线谱编辑功能分为多个层级：

| 层级 | 选中后操作 |
|------|------------|
| 曲谱末尾 | 添加复谱表 |
| 复谱表 | 前/后添加复谱表、添加单谱表、删除 |
| 单谱表 | 前/后添加单谱表、添加小节、删除 |
| 小节 | 前/后添加小节、添加音符/休止符（前/上/后）、添加 volta、调号、拍号、谱号、小节线、删除 |
| 音符头 | 上下拖拽改 region、改时值、改 beamType、添加/改变音号、添加/改附点、删除 |
| 连音线 | 选中起始音符 → 再选结束音符添加；选中连音线可拖拽、删除 |
| 变音号/附点 | 删除 |
| Volta | 拖拽、删除 |

## 四、技术要点备忘

1. **targetId 与数据映射**：VDom 的 `targetId` 对应 MusicScore 中各类实体的 `id`（grandStaff.id、measure.id、note.id、notesInfo.id 等），编辑时需通过 `targetId` 反向找到数据并做增删改
2. **slotData**：slot 节点的 `slotData` 为 `readonly` 的 GrandStaff/SingleStaff/Measure，可用来获取上下文
3. **diffAndMergeVDom**：musicScore.vue 中 `import { diffAndMergeVDom } from '@/render/update'` 但未使用；`render/update.ts` 已删除，可能需清理该导入或恢复 update 模块
4. **数据不可变**：编辑时应对 `musicScoreData` 做结构化更新（如克隆后替换），以触发 watch 重新计算 vDom

## 五、编辑扩展实现思路

1. **dr-edit 插件**：作为 deciphony-test 的 dr-plugins 之一，提供编辑模式、选中状态、快捷键等
2. **选中 → 数据定位**：建立 `targetId → 路径` 的查找函数，如 `findByTargetId(data, targetId)` 返回 `{ parent, index, node }`
3. **指针交互**：在 pointerdown 识别 targetId → 设置选中；在 pointermove（拖拽）根据 delta 更新 region 等；在 pointerup 结束拖拽
4. **命令/操作**：每个编辑动作封装为对 `musicScoreData` 的纯函数更新，便于撤销/重做（若需）
