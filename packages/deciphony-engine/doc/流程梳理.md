### 存档功能

首先是各种节点组成的 json 数据。它是 `game.vue` 接收到可以渲染出来的格式，也就是游戏的本体。

存档功能则靠 `sceneId` 和 `extraData` 数据。`game.vue` 额外接受一个 `extraData`，作为存储数据的容器，这个通常配合条件节点来影响游戏的走向。

`sceneId` 可以控制游戏从哪个场景开始。

---

### extraData 双向同步

`extraData` 由宿主传入，存放好感度、旗标等运行时玩家状态。游戏本体（节点图）始终来自 `props.gameData`，不进存档。

| 方向 | 机制 |
|------|------|
| 外部 → 引擎 | `props.extraData` / `v-model:extra-data` → store；条件节点 `runCode(nodeMap, extraData)` 即时生效 |
| 引擎 → 外部 | DataChange 行为修改 `extraData` → `emit('update:extraData')`；切换场景 → `emit('autoSave', { sceneId, extraData })` |

**条件节点示例**（选项可见性等）：

```js
return extraData.favor >= 10
```

**DataChange 行为示例**（选项选中后改好感度）：

```js
extraData.favor = (extraData.favor || 0) + 5
```

**宿主监听自动存档：**

```vue
<GameView
  :game-data="engineNodes"
  v-model:extra-data="saveData"
  :scene-id="sceneId"
  @auto-save="onAutoSave"
/>

<script setup>
function onAutoSave({ sceneId, extraData }) {
  // 持久化 { sceneId, extraData }
}
</script>
```

节点偏移等视觉效果应通过 **Animation 行为** 实现，不应写入存档。

### 行为定义（ActionTypeEnum）

| 行为                      | 说明                    |
|-------------------------|-----------------------|
| ShowImage / HideImage   | 显示 / 隐藏图片节点           |
| ShowVideo / HideVideo   | 显示 / 隐藏视频节点           |
| PlayVideo / StopVideo   | 播放 / 停止视频             |
| PlayAudio / StopAudio   | 播放 / 停止音频             |
| ShowFilter / HideFilter | 显示 / 隐藏滤镜节点           |
| ShowCustom / HideCustom | 显示 / 隐藏自定义节点          |
| Animation               | 动画行为（写入 animationMap） |
| Combined                | 组合式行为                 |
| Next                    | 场景或对话节点的跳转            |
| Custom                  | 自定义行为                 |
| DataChange              | 数据修改行为                |

行为节点会绑定目标节点的 id。支持动画的节点会 watch `animationMap`。

幕布不属于上述行为：幕布动画在 `curtainNodeBox` 的 `onMounted` 里执行；场景切换时由 `SceneNode.endCurationId` 自动触发（见下文）。

---

### 幕布机制

切换场景（`startScene`）时，在更新 `curSceneId` **之前**：

1. 读取**上一个场景**的 `endCurationId`
2. 将对应 Curtain 节点写入 `viewerCurtainNodeMap` 并渲染
3. 等待幕布动画时长的一半后再清空视图、进入新场景

因此不需要单独的「激活幕布」行为；历史上曾设计 `ActiveCurtain`，已从枚举中移除。
