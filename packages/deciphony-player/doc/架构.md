### MPlayer

节拍器
MPlayer.setTimeSignature(beat,beatCount) // 4 2
MPlayer.start(n) // 从beatCount的第几拍开始，不传会接着前一拍的下一拍继续
MPlayer.pause() // 会播完当前拍后停止，恢复播放会等这个拍播完再恢复
MPlayer.stop()
// 循环
MPlayer.loop = true

但是要的是播放开始时间那种，而不是一个播放完播下一个

精准播放

检查器，每50ms一次，计算未来100ms的播放音符，多出的50ms是为了防止js阻塞

第一次，添加了7个音符
执行source.start(currentTime+offset)进行准确播放

第二次, 实际过了55ms, 那么，对未来100ms,也就是55~155毫秒的音符
重复上述的source.start(currentTime+offset)
实际因为0~100的已经做过了，所以对100~155毫秒区间内的音符执行操作

但是，期间用户执行变速，变调操作该怎么办，这一点tonejs的对过去执行方式是可以实现的，但我是对未来执行，不知道用户的操作

任何改变，都会影响已经start的音符，只会对之后的音符操作
所以改变会有100ms的延迟，这个无伤大雅
