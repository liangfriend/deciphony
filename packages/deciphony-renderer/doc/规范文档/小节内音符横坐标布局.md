# 小节内音符横坐标布局（重构规格）

> **状态**：已实现（线谱 + 简谱共用 `render/layout/measureColumnLayout.ts`；简谱加时线虚拟 onset 见 §3.3）  
> **关联**：`renderSymbol.ts`、`note.ts` / `standardStaff/render/utils/note.ts`（`getNoteWidthRatio`）、`numberNotation/render/layout/measureColumnLayoutAdapter.ts`、`musicScoreToVDomImpl.ts`（连谱 `linkedStaff`）
> **替代**：原「按槽位序号累加 `widthRatio / totalNoteRatio`」的小节内定位方式

---

## 1. 不变部分

### 1.1 小节总宽（`widthRatioForMeasure`）

- **不变**。仍用现有逻辑：各小节（连谱模式下为复谱表级统一列宽）按 `widthRatioForMeasure` 分配横向空间。
- 实现参考：`getMeasureWidthRatio`、`musicScoreToVDomImpl` 中 `measureWidths` / `totalWidthRatioForMeasure`。

### 1.2 定宽 / 变宽三分法

小节内横向仍分为：

| 区域 | 内容 | 宽度算法 |
|------|------|----------|
| 前置（定宽） | clef_f、keySignature_f、timeSignature_f、barline_f 等 | 皮肤 `w` 累加 |
| 变宽 | 音符、休止符 | 本文档 |
| 后置（定宽） | barline_b、clef_b、keySignature_b、timeSignature_b 等 | 皮肤 `w` 累加 |

```text
变宽区宽度 noteDomainW = measureWidth - prefixW - suffixW
```

### 1.3 音符级 widthRatio

- **五线谱**：`standardStaff/render/utils/note.ts` → `getNoteWidthRatio`（符头皮肤 + 变音号、附点、倚音区、音符前谱号等附属）。
- **简谱**：`numberNotation/render/utils/note.ts`；皮肤包仅存 **四分音符档** `widthRatio`，`chronaxie ≥ 64` **不乘时值系数**；二分/全音符的「多出来的四分宽度」由 **加时线虚拟 onset 列**（§3.3）表达，而非把单列 `widthRatio` 乘 2/4。

## 2. 核心思路

1. 用 **chronaxie 累积时值** 作为小节内的时间网格索引（不是槽位序号）。
2. 每个 ** onset 格**（有音符/休止符起始的位置）占一列宽度；列宽由 **默认基础 ratio + 该格音符 `getNoteWidthRatio`** 决定。
3. 各列宽度按 **列 widthRatio 占全小节列 ratio 总和的比例**，分配 `noteDomainW`。
4. 音符 **x** 由「前面各列宽度之和 + 列内锚点」确定：**简谱居左**（`slotX = columnStart + graceBeforeW`），**线谱居中**（列宽内按 `referenceW`、倚音宽居中）。
5. **连谱模式**：同一复谱表、同一小节列、**相同 onset 格** 的列 `widthRatio` 在各单谱表间 **取算术平均**（不是相加）；`totalChronaxie` 取同列各谱表累加值的 **max**（§3.1）。所有单谱表共用同一份「每格 widthRatio / 每格宽度」。

这样既保证 **同 onset 跨谱表对齐**，又能在某一谱表该格有变音号等附加宽时，通过 `getNoteWidthRatio` 抬高列宽，使各谱表同列一起略右移（与 MuseScore 等行为一致）。

## 3. 常量与术语

| 名称 | 建议值 / 含义 |
|------|----------------|
| `DEFAULT_CELL_BASE_WIDTH_RATIO` | 默认 **10**（每 onset 格的基础 widthRatio，与音符 ratio 相加） |
| `totalChronaxie` | 小节内 **实际 slot** 时值累加（见 §3.1）；用于时间轴尺度，**不**用于等分像素 |
| `onset` | 某 slot 在小节内的起始时间索引 = 其前面所有 slot 的 **布局时值** 之和（简谱见 `getSlotLayoutChronaxie`，含附点延长） |
| `onset 格 / 列` | 以 `onset` 为键的一列；列宽由 `DEFAULT + widthRatio` 登记后按比例分配 `noteDomainW` |

### 3.1 `totalChronaxie` 怎么算（重要）

**不是**把小节固定分成 256 份，也 **不是** 按拍号（`timeSignature`）推导。

- 拍号只是展示/编辑辅助，**不限制**用户在小节里放多少音符；布局只认 `measure.notes` 里真实存在的 slot。
- `totalChronaxie` = 该小节内所有 slot 的 **布局时值** **相加**（简谱用 `getSlotLayoutChronaxie`：含附点延长；虚拟加时线 onset **不**重复累加）。
- 数据模型里 `chronaxie` 仍取标准档（附点二分 = `128` + `augmentationDot`）；**onset 推进**按附点延长后的时值（附点二分 → **192**）。
- 例：三个四分 → 64×3 = **192**；`4 4 2` → 64+64+128 = **256**；`| 5 - 5 5 |` → **0、128、192**；附点二分 `2 -.` 单独占 **192**，下一 slot 从 **192** 起。

**单谱表**：该谱表当前小节的 `totalChronaxie` 即上式累加结果。

**连谱**（同一复谱表、同一小节列 `mi`）：对位上的每个单谱表各算一个累加值，取 **最大值** 作为本小节列统一的 `totalChronaxie`：

```text
totalChronaxie = max over staves ( Σ getSlotChronaxie(slot) )
```

较短谱表在尾部可能出现无符头的 onset 列占位，但 **同 onset 仍与最长谱表对齐**。

### 3.2 onset 与列（非逐 tick 切像素）

- `onset` 落在 `[0, totalChronaxie)` 的时间索引上，由 slot 顺序与前面 slot 时值累加得到。
- 布局 **不会** 把 `noteDomainW` 机械切成 `totalChronaxie` 个等宽像素格。
- 在 **有 widthRatio 登记的 onset** 处设 **列**（含简谱加时线虚拟 onset，§3.3）；列宽由 §4.2 / §5.3 的比例分配 `noteDomainW`：

```text
cellWidth[g] = cellWidthRatio[g] / Σ cellWidthRatio × noteDomainW
```

示例：

| 小节内容 | totalChronaxie | 符头 onset（slot 起点） |
|----------|----------------|-------------------------|
| `4 4 2` | 256 | **0、64、128** |
| `5 - 5 5`（二分+四分+四分） | 256 | **0、128、192** |
| 四个四分 | 256 | **0、64、128、192** |

### 3.3 简谱加时线虚拟 onset（仅 numberNotation）

线谱无加时线，不适用本节。简谱通过 adapter 的 `getExtraOnsetRatios`，在 **符头 onset 之外** 为每条加时线再登记一列，使连谱时与下方四分音符 **同 onset 对齐**。

设皮肤四分档 `widthRatio = x`，变音 `y`，附点 `z`（均来自皮肤或符号自身 `widthRatio`）。

**一个 slot 仍只占一次 onset 推进**（全音符 `chronaxie=256` 只把下一个 slot 的起点推进 256）。虚拟列偏移为 **+64、+128、+192…**（每到下一个四分边界一条加时线）。

对位于符头 onset `g` 的全音符 `#5 - - -.`：

| onset | 登记内容（不含 DEFAULT 10） | 说明 |
|-------|----------------------------|------|
| `g` | `x + y` | 符头 + 变音等；**不含附点** |
| `g+64` | `x` | 加时线列，仅皮肤档 |
| `g+128` | `x` | 加时线列，仅皮肤档 |
| `g+192` | `x + z` | **最后**一条加时线列 + 附点 |

二分无附点仅多登记 **`g+64`** 一列。四分无虚拟列，附点与符头同在 `g`。

### 3.4 简谱二分附点特例（第一个附点 → 加时线）

**仅 `chronaxie = 128`（二分）** 且带 `augmentationDot` 时适用；四分、全音符等仍按 §3.3 常规规则（附点在末条加时线列或符头旁）。

简谱视觉上 **不把第一个附点画成点**，而是 **多画一条加时线**，并把列布局 **重算为三等分**（三个 onset 列）：

| 数据 | 视觉 | 虚拟 onset（相对符头 `g`） | 附点渲染 |
|------|------|---------------------------|----------|
| `2 -`（128，无附点） | `2 -` | `g+64`（1 条加时线） | 无 |
| `2 -.`（128，1 附点） | `2 - -` | `g+64`、`g+128`（2 条加时线） | **无**（首附点已消耗） |
| `2 -..`（128，2 附点） | `2 - -.` | `g+64`、`g+128` | 末列 1 个点（`count-1`） |
| `2 -...`（128，3 附点） | `2 - -..` | `g+64`、`g+128` | 末列 2 个点 |

**widthRatio 登记**（设皮肤四分档 `x`，变音 `y`，剩余附点 `z`）：

| onset | `2 -.` | `2 -..` |
|-------|--------|---------|
| `g` | `x + y` | `x + y` |
| `g+64` | `x`（首附点 **不** 记入，作加时线） | `x` |
| `g+128` | `x` | `x + z`（`z` 为扣除首附点后的附点宽） |

规则摘要：

1. 第一个附点 **永不** 进入附点 `widthRatio`，也 **不** 渲染为附点符号。
2. 第一个附点使加时线条数 +1，并多登记一列虚拟 onset（按皮肤 `x` 计宽，与常规加时线相同）。
3. 其余附点（若有）仍在 **最后一条加时线** onset 列登记 `z` 并渲染。
4. slot **onset 推进**仍为附点延长时值（`2 -.` → **192**），与下方四分 onset **64 对齐**。

实现：`getAddLineOnsetOffsets`、`getRenderableAugmentationDot`、`getSlotLayoutChronaxie`（`note.ts`）。

**连谱示例**（两声部，皮肤 x=20，变音 z_acc=30，附点 z=40）：

```text
声部1:  4   4   4   4.
声部2:  #4  -   -   -.

onset 0:   (10+20) 与 (10+20+30) → 平均
onset 64:  (10+20) 与 (10+20)     → 平均
onset 128: (10+20) 与 (10+20)     → 平均
onset 192: (10+20+40) 与 (10+20+40) → 平均
```

**渲染**：符头 x 在 onset `g` 列内 **居左**；第 k 条加时线取 onset `g + 64×(k+1)` 列的 **列起点 x**；附点锚在 **最后一条加时线** 所在 onset 列（无加时线则在符头右侧）。

实现：`numberNotation/render/utils/note.ts`（`getNoteHeadColumnWidthRatio`、`getNumberNotationExtraOnsetRatios`）、`measureColumnLayoutAdapter.ts`。

## 4. 单谱表算法

### 4.1 收集 onset 列

遍历 `measure.notes` 顺序，维护累积时值 `acc`：

```text
onset(slot[0]) = 0
onset(slot[i]) = onset(slot[i-1]) + layoutChronaxie(slot[i-1])
```

简谱 `layoutChronaxie` = `getSlotLayoutChronaxie`（含附点延长）；线谱 = `getSlotChronaxie`。

每一 slot 在其 `onset` 处占一列（同一 measure 内 onset 不重复）。

### 4.2 每列 widthRatio

**五线谱**（onset = `g` 上的 slot）：

```text
cellWidthRatio[g] = DEFAULT + getNoteWidthRatio(slot, skin, measureHeight)
```

**简谱**：

- 符头 onset `g`：`DEFAULT + getNoteHeadColumnWidthRatio(note)`（含皮肤 + 变音/倚音等；有加时线时 **不含附点**）。
- 每条加时线虚拟 onset `g+64`、`g+128`…：由 `getExtraOnsetRatios` 追加登记；中间列为 `DEFAULT +` 仅皮肤档，**最后一列**可再加附点 `z`。

### 4.3 列宽与总宽

```text
totalCellWidthRatio = Σ cellWidthRatio[g]   （对该小节所有 onset 列求和）

cellWidth[g] = cellWidthRatio[g] / totalCellWidthRatio × noteDomainW
```

列的基础部分来自 `DEFAULT_CELL_BASE_WIDTH_RATIO`（10），再叠加该格 `getNoteWidthRatio`；各列按 ratio 总和 **比例** 瓜分 `noteDomainW`，与 `totalChronaxie` 无「一等分像素」关系。

### 4.4 音符 x

列起始 x：

```text
columnStart[g] = noteDomainStartX + Σ cellWidth[g']  （对所有 g' < g 的 onset 列求和）
```

`noteDomainStartX = measureX + prefixW`（与现 `renderSymbol` 一致）。

列内符头 x：

- **线谱**：列宽内居中（`referenceW`、倚音 before/after、`graceBeforeW`）。
- **简谱**：列起点 + `graceBeforeW`，**居左**；加时线取对应虚拟 onset 列起点 x。

## 5. 连谱模式（`linkedStaff = true`）

### 5.1 适用范围

- 同一 **复谱表**（`grandStaff`）内、同一 **小节列索引** `mi` 的所有 `staff.measures[mi]`。
- 小节 **外框宽度** 已统一（现有 `measureWidths[mi]`）；本节只统一 **小节内各 onset 列的 widthRatio 与 cellWidth**。

### 5.2 统一 `totalChronaxie` 与合并 onset 格

对该小节列下所有单谱表：

1. 各谱表分别累加 slot 时值，取 **max** 作为本列统一的 `totalChronaxie`（§3.1）。
2. 各谱表分别按 §4.1 计算每条 slot 的 `onset`（仍基于 **本谱表** 的 slot 顺序与前时值累加）。
3. 取 **所有谱表 onset 的并集** 作为列索引集合（例：`{0, 64, 128}`）。

### 5.3 每格 widthRatio 取平均

对每个 onset 格 `g`：

1. 对每个在该 `g` **有 widthRatio 登记** 的单谱表 `s`（含简谱虚拟加时线 onset），计算 `cellWidthRatio_s[g]`。
2. 连谱列 ratio（**算术平均**，不是相加）：

```text
cellWidthRatio[g] = average( cellWidthRatio_s[g] )   // 仅对有 slot 的谱表求平均
```

**示例**（`DEFAULT = 10`；四分 `getNoteWidthRatio = 10`，二分 `= 20`）：

```text
谱表1:  4    4    2     → onset: 0, 64, 128
谱表2:  2         2     → onset: 0, 128

onset 0:   谱表1 → 10+10=20，谱表2 → 10+20=30  → 平均 (20+30)/2 = 25
onset 64:  仅谱表1 → 10+10=20                    → 20（不平均）
onset 128: 两谱表均为 10+20=30                   → 平均 30
```

### 5.4 共用列宽

对并集 onset 集合，用 §4.3 算 `cellWidth[g]` 与 `columnStart[g]`。**同一复谱表、同一小节列的所有单谱表共用同一份** `cellWidth` / `columnStart` 表。

各谱表 `renderSymbol` 只读该表定位；`prefixW` / `suffixW` 仍可各谱表不同，但 **变宽区列网格一致**。

### 5.5 空列行为

某谱表在 onset `g` 无音符，但其他谱表有 → 该谱表在 `g` 仍保留 **相同 `cellWidth[g]`** 的横向占位（时间对齐）；该格无符头，不渲染 slot。

## 6. 与旧实现的差异

| 项目 | 旧实现 | 新规格 |
|------|--------|--------|
| 横轴索引 | 槽位序号 `i` | chronaxie **onset** |
| 列宽 ratio | 每槽 `getNoteWidthRatio` 直接参与 `totalNoteRatio` | 每 **onset 列** `10 + getNoteWidthRatio` |
| 连谱 | 仅小节外宽统一；槽位比例各谱表独立 | `totalChronaxie` 取同列 **max**；onset 并集 + 列 ratio **跨谱表平均**；列宽共用 |
| 简谱长时值 | — | 皮肤按四分档；**虚拟加时线 onset** 复制皮肤列；附点在末条加时线 onset |
| 简谱二分附点 | — | **首附点 → 加时线**；三等分 onset；见 §3.4 |
| 列内对齐 | — | 简谱 **居左**；线谱 **居中** |
| 变音号影响 | 主要加在本谱表该槽 ratio | 加在本谱表该格 ratio → 连谱时抬高共享列宽 → 各谱表同列后移 |

## 8. 已知细节

1. **仅单谱表参与的 onset 格**：平均退化为单值（见 §5.3 示例 onset 64 → 20）。
2. **连谱空列**：某谱表在 onset `g` 无符头，但并集含 `g` → 仍占相同 `cellWidth[g]`（§5.5）。
3. **倚音**：宽计入符头 onset 的 `getNoteHeadColumnWidthRatio` / 线谱 `getNoteWidthRatio`，挂在宿主列内渲染。
4. **简谱多声部 `notesInfo[]`**：列布局按 slot 最大 `chronaxie` 登记虚拟 onset；各层减时线/beam 按层独立渲染（见 `floorSpan`）。
