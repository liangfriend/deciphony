# 附属符号渲染规则

四类附属符号（单音符 / 双音符 / 单小节 / 双小节）统一由 **规则常量 + 渲染引擎** 驱动，不在渲染函数里为每种符号写 `if/switch`。

## 规则文件

`src/constants/renderRules.ts`

| 常量 | 数据挂载点 |
|------|------------|
| `SingleNoteAffiliatedSymbolRenderRule` | `NotesInfo.affiliatedSymbols`（线谱）/ `NoteNumber.affiliatedSymbols`（简谱） |
| `DoubleNoteAffiliatedSymbolRenderRule` | `musicScore.affiliatedSymbols`（连音线等） |
| `SingleMeasureAffiliatedSymbolRenderRule` | `measure.affiliatedSymbols` |
| `DoubleMeasureAffiliatedSymbolRenderRule` | `musicScore.affiliatedSymbols`（volta 等） |

偏移量单位均为 **× measureHeight**。

## 渲染策略 `kind`

| kind | 说明 |
|------|------|
| `skin` | 查皮肤包，生成 `tag: 'affiliation'` + `skinKey` 的 VDom，由 `Group` 组件绘制 |
| `slur` | 双音符：锚定起止音符头，写入 `special.slur`，由 `Slur` 组件绘制 |
| `volta` | 双小节：锚定起止小节，写入 `special.volta`，由 `Volta` 组件绘制 |

未实现的符号**不写规则项**即可，引擎遇到无规则项时静默跳过。不要批量生成占位规则。

## 渲染入口

`src/render/affiliated/engine.ts`

| 函数 | 调用时机 |
|------|----------|
| `renderSingleNoteAffiliatedSymbols` | 音符头 VDom 写入 `idMap` 之后（`renderSymbol` 内） |
| `renderSingleMeasureAffiliatedSymbols` | `renderSymbol` 之后、`processBeam` 之前 |
| `renderMusicScoreAffiliatedSymbols` | 全谱布局完成后（slur / volta / 双音符皮肤类） |

**反复符号**（Coda、DC、Fine 等）不在 `affiliatedSymbols` 中，走 `measure.startRepeat` / `endRepeat`，见 [反复符号渲染](./反复符号渲染.md)。

## 双音符 / 双小节分发

`slur` 与 `volta` 在枚举中同为数值 `0`，引擎按 **锚点 id 在 idMap 中的 tag** 区分：

1. `startId` 对应 `measure` → 双小节规则（volta）
2. 否则 `startId` 对应 `noteHead` → 双音符规则（slur 等）

## 单音符规则写法

在 `SingleNoteAffiliatedSymbolRenderRule` 中**手动**添加需要的项，例如：

```typescript
[SingleNoteAffiliatedSymbolNameEnum.Staccato_above]: {
    kind: 'skin',
    skinKey: 'staccato_above',
    vertical: -1 / 8,
    horizontal: 0,
},
```

皮肤尚未定义时 `resolveSkinSize` 返回 null，不输出 VDom。

## Vue 层

`musicScore.vue`：`affiliation` 节点在 `<g>` 内按 `special` 分发；`slot` 之后 `v-else` 走 `Group`（皮肤类附属符号）。
